# v0.1.123 - 恢复播放器自动旋转全屏

## 更新内容

- 恢复视频播放页的设备方向监听，横屏时自动进入全屏，不再必须手动点击全屏按钮。
- 自动横屏仍使用 AVKit/AVPlayerViewController 容器，继续复用同一个 AVPlayer，避免重建播放实例。
- 竖屏时自动退出全屏并回到内嵌播放器，播放进度和播放状态保持不变。
- 自动全屏进入时会把同一个 AVPlayer 从内嵌容器搬到全屏容器，退出后再搬回，避免两个播放器容器争抢画面。
- 同步恢复弹幕承载层、手势层和 videoBounds 区域，横竖屏切换后覆盖层仍贴合真实视频画面。
- 支持系统全屏按钮和自动旋转两种路径，并增加保护避免重复弹出全屏。

## 打包说明

- GitHub Actions 会构建 Release 配置的未签名 IPA。
- 生成的 `ccbili-unsigned.ipa` 会作为 Release 附件上传。
- 未签名 IPA 需要自行重签名后安装到真机。

## 验证建议

- 竖屏进入视频播放页并开始播放。
- 直接把手机旋转到横屏，确认自动进入横屏全屏。
- 旋回竖屏，确认自动退出全屏并回到内嵌播放。
- 验证播放不中断，进度不跳变，弹幕/手势层不重复添加。
- 再测试手动点击系统全屏按钮，确认与自动旋转路径不冲突。
