# v0.1.122 - 播放器横屏与性能优化

## 更新内容

- 播放器改为复用同一个 AVPlayer 实例，横竖屏切换不再销毁或重建播放对象。
- 使用 AVPlayerViewController 原生全屏过渡，减少手写旋转和额外窗口带来的卡顿、掉帧和发热。
- 新增横屏播放器容器，进入全屏时优先支持 landscapeLeft / landscapeRight，退出后回到竖屏内嵌播放。
- 弹幕承载层和手势层移动到 AVPlayerViewController.contentOverlayView，并随 videoBounds 更新真实视频画面区域，避免覆盖黑边。
- 保留系统原生播放控制条，并关闭外层重复自定义控制层，避免双控制条和触摸冲突。
- 手势层支持双击播放/暂停、长按倍速、横向进度拖动、左右区域亮度/音量调节。
- 播放状态、倍速、进度和画中画能力在全屏切换前后保持一致。
- 降低播放进度刷新频率和诊断刷新开销，减少视频页滑动时的 SwiftUI 状态更新压力。
- 评论区改为按需加载和懒渲染，头像使用远程图片缓存，降低视频页滚动掉帧。
- 页面背景继续使用 iOS 系统分组灰色，保持搜索页/详情页视觉一致。

## 打包说明

- GitHub Actions 会构建 Release 配置的未签名 IPA。
- 生成的 `ccbili-unsigned.ipa` 会作为 Release 附件上传。
- 未签名 IPA 需要自行重签名后安装到真机。

## 手动验证建议

- 竖屏进入视频页，开始播放后点击系统全屏按钮进入横屏。
- 退出全屏后确认回到竖屏内嵌播放，播放不中断。
- 反复横竖屏切换，确认没有重复 overlay、控制条遮挡或明显内存增长。
- 验证双击、长按、滑动亮度/音量/进度手势仍可用。
- 在视频页和评论区快速滑动，确认掉帧和发热有所缓解。
