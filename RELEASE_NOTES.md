# v0.1.125 - 优化横竖屏切换播放连续性

## 更新内容

- 横竖屏切换时保持自动播放意图，不再因为 AVKit 瞬时 waiting 状态误判成暂停。
- 进入全屏、退出全屏和系统全屏回调期间都会主动续播，播放中的视频切换方向不再停住。
- 切换过程中临时锁定播放状态显示，避免播放按钮和进度条出现短暂“暂停/卡一下”的抖动。
- 播放进度刷新从 1 秒缩短到 0.5 秒，并在主线程时直接更新，减少切换后的进度延迟感。
- 弹幕/手势层的 videoBounds 更新增加合并和去重，降低横竖屏动画期间重复刷新导致的卡顿。

## 打包说明

- GitHub Actions 会构建 Release 配置的未签名 IPA。
- 生成的 `ccbili-unsigned.ipa` 会作为 Release 附件上传。
- 未签名 IPA 需要自行重签名后安装到真机。

## 验证建议

- 竖屏播放视频，旋转到横屏，确认播放不中断且不会自动暂停。
- 横屏播放一段时间后旋回竖屏，确认进度条连续前进，没有明显停顿。
- 反复横竖屏切换 5 次以上，确认按钮状态、弹幕和手势层正常。
- 点击系统全屏按钮和自动旋转路径都测试一次，确认体验一致。
