# v0.1.132 - 优化横屏呈现时序

## 更新内容

- 修复手机旋转到横屏时，播放器先以竖屏全屏出现、再旋转成横屏的过渡异常。
- 自动横屏路径改为先锁定目标横屏方向，等待窗口方向真正变为横屏后，再呈现临时全屏播放器。
- 竖屏内嵌播放器保持竖屏方向，避免 AVKit 在传感器变化时抢先把原播放器拉入竖屏全屏。
- 临时横屏播放器继续复用同一个 `AVPlayer`，不重建播放源，播放进度、缓冲和声音状态保持连续。
- 只有在临时全屏控制器即将呈现时，才临时断开竖屏播放器的 `player`，减少黑屏和空白闪烁窗口。
- 手动全屏按钮路径同步收窄到当前目标横屏方向，避免开放三方向导致先竖屏呈现。

## 原因说明

之前自动旋转时，临时横屏播放器的 `present` 发生得太早：系统窗口还停留在竖屏几何状态，AVKit 会先按竖屏尺寸完成全屏呈现，然后才响应横屏方向更新，所以会看到“竖屏全屏 → 横屏”的二段式跳变。

本版本把流程调整为“锁定播放器横屏方向 → 请求系统窗口横屏 → 确认窗口已横屏 → 呈现临时全屏播放器”，并让内嵌播放器在页面内只声明竖屏，防止原竖屏控制器抢跑进入全屏。

## 验证建议

- 竖屏进入视频详情页并开始播放。
- 不点全屏按钮，直接旋转手机到横屏，确认不会先出现竖屏全屏画面。
- 横屏播放时确认画面、声音、进度连续，不重播、不跳进度。
- 从横屏旋回竖屏，确认回到内嵌播放器且弹幕、手势层仍正常。
- 反复横竖屏切换 5 次以上，确认无黑屏、无闪退、无重复 overlay。
